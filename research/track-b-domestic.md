# Track B — 국내 사례 조사 (이미 존재하는 복지 알림·갱신·자격 서비스)

조사일: 2026-06-22 · 조사자: Track B (국내 사례) · 모든 출처 URL·발행처·확인시점 명기

## 요약 (한 줄)
국내에는 "내가 받을 혜택을 찾아 알려주는" 서비스(복지멤버십·보조금24·국민비서)는 잘 갖춰져 있으나, **이미 받던 혜택이 끊기는 것(자격 탈락 조기경보)**, **갱신·서류 누락을 끝까지 챙기는 체크리스트**, **어르신 음성 점검**의 세 영역은 거의 비어 있다. 우리 킥 중 **K1(자격변동 조기경보)**과 **K3(어르신 음성 점검)**이 국내 공백이 가장 크다.

## 1. 복지로 '복지멤버십(맞춤형 급여 안내)'

**무엇을 하나**
- 개인·가구의 연령, 가구구성, 경제상황(공적자료)을 기준으로 받을 수 있는 복지서비스를 자동으로 찾아 **안내**하는 제도.
- 2025/2026 기준 **총 163종(지자체 포함)** 급여를 안내. 생계급여, 장애수당, 디딤씨앗통장, 난임부부 시술비, 고교 학비, 가정양육수당, 아이돌봄, 청년월세, 에너지바우처 등.
- 알림 채널: **문자·이메일 → 2026년 4월부터 카카오톡까지 확대.** 카톡 알림 내 "복지 서비스 확인하기" 버튼으로 신청 화면 연결.

**자격 변동·갱신·서류 누락을 다루나**
- **자격 변동(신규 진입형): 일부 다룬다.** "신청인의 공적자료를 기반으로 변동 사항 발생 시 시스템이 자동 판정", "출산·연령 도래 등 가구원 변동 발생 시 (새로) 수급 가능한 급여 안내".
- 그러나 방향이 **'새로 받을 수 있게 된 급여 안내'(상향)**에 맞춰져 있고, **'기존에 받던 급여가 끊길 위험(하향/탈락) 사전경보'는 명시적으로 다루지 않는다(미확인 → 사실상 공백).**
- 갱신 시기·서류 누락 체크리스트는 제공하지 않음(미확인).

**사용자 마찰 / 한계**
- **사전 가입 필수**: 복지로 또는 주민센터에서 본인이 직접 등록해야 안내를 받는다 (가입 안 하면 아무 안내 없음).
- **신청 책임 본인**: 안내를 받아도 실제 신청 절차는 본인이 완료해야 한다.
- **포괄성 제한**: 전체 약 5,000개 복지서비스 중 **163종만** 자동 안내.
- 어르신 대상 음성/대면 점검 기능 없음(웹·앱·카톡 텍스트 중심).

## 2. 정부24 '보조금24 / 국가보조금 맞춤형 서비스'

**무엇을 하나**
- 정부·지자체·공공기관·교육청의 보조금/혜택을 개인 자격정보(나이·소득·가구·거주지·직업 등)로 맞춤 선별해 한 곳에서 **조회·신청**.
- 행정안전부가 운영하며 '보조금24' API를 카카오뱅크·카카오톡지갑 등 민간에 개방.

**자격 변동·갱신·서류 누락을 다루나**
- **알림 = 국민비서 연동.** "신청 시기가 도래하거나, 자격 변동 시 미리 혜택을 찾아 국민비서를 통해 알림". 새 혜택 추가, **신청 마감일 임박**, 지급 일정 알림 제공.
- 즉 **'마감 임박' 알림은 있으나**, 이 역시 **신규 신청 기회** 중심. 기존 수급의 탈락/중단 경보, 서류 누락 보완 체크리스트는 명시 안 됨(미확인 → 공백).

**사용자 마찰 / 한계**
- 정부24 회원·로그인·이용동의 필요. 맞춤안내 조회는 회원 대상.
- 텍스트·웹/앱 중심 → 디지털 취약 어르신 접근성 한계.
- 알림은 받아도 "이 조건을 정말 충족하는지" 최종 판단·신청은 본인 몫.

## 3. 국민비서 '구삐' (알림서비스)

**무엇을 하나**
- 정부의 통합 알림 허브. 은행앱(국민·농협·신한·우리·하나·기업 등)·네이버·카카오·토스·PASS 등 **20개 이상 채널**로 푸시.
- 복지멤버십·보조금24 알림의 실제 **전달 백엔드** 역할.

**자격 변동·갱신·서류 누락을 다루나 — (가장 근접한 기존 사례)**
- **갱신 알림 일부 보유**: 운전면허 적성검사 갱신(연 3회), 여권 만료 6개월 전, 무선국 재허가, 농업경영체 등록 만료, '나의생활정보'(여권만료·자동차검사·종소세·재산세 등 6종).
- **자격 변동 통지 일부**: 국민연금 지역가입자 월별 자격변동 통지, 장애정도 심사결과 즉시 알림.
- **서류 보완 알림 일부**: 세움터(보완요청/보완완료/재보완요청), 부동산 실거래조사 증빙 제출 안내, 문서24 접수/실패.
- 복지 상담 분야 확대: 긴급복지, 기초생활보장, 보육·아동·출산, 노인학대, 장애인·정신건강 정책 상담.

**한계**
- 위 갱신·자격변동 알림은 **개별 제도별로 흩어진 단발성 통지**이지, **'어르신 수급자의 받던 복지급여 전체를 묶어 탈락 위험을 사전 경보'하는 통합 기능은 아님.**
- 서류 보완 알림도 건축(세움터)·문서 행정 위주이고 **복지급여 서류 누락 체크리스트는 아님**(미확인).
- 텍스트 푸시 중심 → 어르신 음성/대면 점검 없음.

## 4. 행복e음 / 사회보장정보시스템

**무엇을 하나**
- **지자체 복지 담당 공무원 전용 행정 내부시스템.** 급여·서비스 대상자의 자격·이력을 통합 관리, 소득·재산 조사 처리.

**시민용인가 공무원용인가 — 공무원용.**
- **일반 국민은 직접 사용 불가.** 사용자 ID는 복지업무 담당 공무원에게만 발급. 시민은 신청 시 공무원이 이 시스템으로 조사·판정.

**시사점(우리에게 중요)**
- 즉 **자격 변동의 '판정 로직과 데이터'는 공무원 백오피스(행복e음)에 갇혀 있고, 당사자(어르신)에게 선제적으로 노출되는 시민용 인터페이스가 없다.** → 우리 K1(조기경보)·K3(음성 점검, 판정은 코드)이 정확히 이 빈틈을 노린다.

## 5. 민간 앱 — '숨은 정부지원금 찾기'

**토스 — 서비스 종료**
- 행안부 공공데이터('보조금24' API) 활용해 금융앱 최초로 자체 앱에서 숨은 지원금 조회 구현(2021.9 오픈).
- **그러나 '숨은 정부지원금 찾기'는 2024년 8월부로 제공 종료.**
- 한계(종료 전): 모든 지자체 지원금 100% 미반영, 연동 데이터만으로 세부조건 충족 정확 판단 어려움, 디지털 취약계층 접근장벽.

**카카오뱅크 — 운영 중**
- '정부지원금 찾기': 수집 정보 기반 받을 수 있는 지원금 선별 추천 + 새 지원금/관심지원금 **신청 마감 임박 알림**. 카카오톡지갑도 보조금24 연동 실시간 알림.

**뱅크샐러드**
- 정부지원금 '추천'은 대출(신용·대환·정부지원·주택·자동차) **금융상품 비교**에 가까움. 복지 자격변동/갱신 알림 특화 기능은 **미확인**.

**공통 메커니즘과 한계**
- 메커니즘: 전부 **행안부 보조금24 API 또는 국민비서 재활용** → 원천 데이터가 같아 차별성은 UX뿐.
- 한계: (1)신규 신청 중심 — **기존 수급 탈락 경보 없음**, (2)세부조건 최종판단·신청은 본인, (3)스마트폰·앱 능숙 전제 → **어르신 배제**, (4)민간은 수익모델 따라 **종료 리스크**(토스 사례).

**핵심 공백 3가지**

3. **어르신 음성 점검** — 전 서비스가 스마트폰·텍스트·로그인 전제 → 디지털 취약 어르신 사실상 배제.

## 7. 우리의 차별점(킥) 후보와 권고

**겹치지 않는 차별점 (최소 1개 → 3개 제시)**
- **K1 자격변동 조기경보**: 기존은 전부 '상향(새 혜택)' 안내. 우리는 소득인정액·재산·가구변동으로 **받던 급여가 끊길 위험을 사전 경고**(하향 방향). 행복e음에 갇힌 판정 로직을 당사자 관점 코드로 재현해 노출 → **국내 직접 경쟁자 없음.**
- **K2 갱신·서류 체크리스트 타임라인**: 산재한 갱신 알림을 **한 어르신의 받는 급여 전체** 기준으로 묶어 "언제·무슨 서류" 타임라인 제공.
- **K3 어르신 음성 점검**: 모든 기존 서비스가 텍스트·앱·가입 전제 → 음성 인터페이스는 **완전 공백.**

**국내에 가장 비어있는(차별화 여지 큰) 방향: K1 > K3 > K2**
- **K1(자격변동 조기경보)**: 직접 대체재가 사실상 없음. "몰라서 끊긴다"는 문제 가설에 정조준. 단, 판정 데이터 확보가 관건(데모는 gov24 10,957 + 복지 5,021 번들 룰 기반으로 시연 가능).
- **K3(음성 점검)**: 타깃이 어르신이라는 점에서 접근성 차별화가 즉시 먹힘. 기존 전무. 발표 5분에서 시각적 임팩트 큼.
- **K2(체크리스트)**: 가치 있으나 국민비서·복지멤버십이 부분적으로 근접 → 차별화 폭은 K1/K3보다 작음. K1을 받쳐주는 보조 기능으로 결합 권장.

**권고 구성**: **K1을 메인 훅(국내 공백 최대) + K3를 전달 채널(어르신 접근성) + K2를 후속 액션(끊기기 전에 뭘 낼지)으로 묶는 단일 시나리오**가 기존 서비스와 한 번도 정면으로 겹치지 않는다.
