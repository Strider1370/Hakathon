# TRACK C — 해외 사례 + 메커니즘 (수급 끊김 방지)

작성: 2026-06-22 / 조사: WebSearch·WebFetch / 대상: 어르신 수급자, 발표 5분

핵심 미션: 기능 복붙이 아니라 **검증된 메커니즘**을 추출해 한국에 이식. 우리 3킥(K1 자격변동 조기경보 / K2 갱신·서류 체크리스트 타임라인 / K3 어르신 음성 점검)에 매핑.

---

## 사례 1. 미국 Medicaid Ex-Parte(자동) 갱신 — "행정적 churn" 차단

- **무엇**: 수급자가 갱신서류를 못 내서 *자격이 있는데도* 탈락하는 'administrative churn(행정적 이탈)' 문제. 미국은 갱신 시 **먼저 정부 보유 데이터(소득·고용 등)로 자격을 자동 확인(ex parte)** 하고, 확인되면 당사자는 아무 행동도 안 해도 갱신됨. 확인 안 될 때만 서류 요청.
- **핵심 메커니즘**: "신청자에게 떠넘기지 말고, 보유 데이터로 먼저 판정 → 통과면 무서류 갱신, 실패분만 사람에게 요청"(부담의 디폴트를 정부 쪽으로 이동).
- **효과 근거**: 주 Medicaid 기관 개입 후 **ex-parte 갱신율 +28%p, 전체 갱신율 +12%p, 절차상 탈락(procedural denial) -9%p** (Health Affairs, 2025). Unwinding 기간 ex-parte 비율이 ~30% → 55%+로 상승(CBPP). Unwinding 동안 2,500만 명 탈락했으나 순감은 1,300만 명 — 상당수가 *자격 있으나 절차로 탈락 후 재등록*.
- **한국 이식 → K1/K2**: 행정정보 연계로 "당사자가 못 챙겨도 시스템이 먼저 점검". 한국 공공데이터로 자격변동을 사전 감지 → 자동 갱신 가능분/서류 필요분 분리. **K1 조기경보의 정당성 근거**이자 **K2 체크리스트는 "ex-parte 실패분"만 사람에게 보내자는 설계 논리** 제공.
- **출처/확인**: Health Affairs 2025 (healthaffairs.org/doi/10.1377/hlthaff.2025.00316), CBPP "Streamlining Medicaid Renewals", ASPE ex-parte data point, medicaid.gov. 확인 2026-06-22.

## 사례 2. 텍스트 리마인더 RCT — 갱신·인터뷰 완료율을 실제로 올린 증거

- **무엇**: SNAP(미국 식품지원) 갱신/인터뷰 마감 전 **문자 알림(nudge)** 의 효과를 무작위대조시험(RCT)으로 측정.
- **핵심 메커니즘**: "마감·다음 행동을 정해진 시점에 짧게 리마인드 → 놓침(missed deadline) 감소". 새 제도가 아니라 *알림 한 줄*로 행정 이탈을 줄임 — 저비용·고효과.
- **효과 근거 (수치)**:
  - **Boulder County, Colorado RCT (2023, n=1,554)**: 문자 3회 발송 그룹이 인터뷰 완료율 **+10.7%p**, 승인율 **+6~7%p(6.3~6.9%)**, 인터뷰를 3~4일 더 일찍 완료. (The Policy Scientist 요약)
  - **GSA OES RCT (2023, n=33,574)**: 문자 + 간편 서류 업로더(비번 제거, 사진 업로드) 링크 → 정보누락 탈락 **-1.1%p (21.1%→20.0%, p=0.012)**. 단, 최종 등록률 차이는 유의하지 않음(기술 오류로 7개 중 2개 메시지만 발송된 한계).
- **한국 이식 → K2/K3**: "타임라인 + 리마인더"가 효과 있다는 **실증 근거**. 단 OES 결과는 *문자만으로는 약하고, 서류 제출 마찰(비번·업로드)까지 같이 줄여야 등록까지 이어진다*는 교훈 — K2 체크리스트가 단순 알림을 넘어 **"무엇을·어떻게 내는지"까지 안내**해야 함을 뒷받침. 어르신 대상이면 문자보다 **음성(K3)** 채널로 동일 nudge.
- **출처/확인**: oes.gsa.gov/results/decreasing-snap-denials, thepolicyscientist.com (Boulder RCT). 확인 2026-06-22.

## 사례 3. Code for America / GetCalFresh — 신청·갱신 마찰 제거 + 서류 제출 지원

- **무엇**: SNAP(캘리포니아 CalFresh) 신청을 10분으로 줄인 디지털 보조 도구. 700만+ 캘리포니아 주민 사용. **진행상황·필요 인터뷰·등록 유지 방법을 문자/이메일로 안내**.
- **핵심 메커니즘**: ①신청·갱신 단계 단순화 ②사용자 선호 채널로 진행상황 가시화(reminder 캠페인) ③**서류 제출 마찰 제거**(사진 업로드, 비번 없는 업로더). "어디까지 했고 다음에 뭘 해야 하는지"를 항상 보여줌.
- **효과 근거**: 10년간 700만+ 사용(Code for America). 서류 제출 지원이 정보누락 탈락을 줄이는 데 기여(사례 2 OES 업로더 결과와 연결). *정량 RCT는 사례 2가 제공*.
- **한국 이식 → K2/K3**: K2 "갱신·서류 체크리스트 타임라인"의 직접 모델. **진행 상태 + 남은 서류 + 마감**을 한 화면(또는 음성)으로. 어르신용은 GetCalFresh의 문자 채널을 **음성 점검(K3)** 으로 치환.
- **출처/확인**: codeforamerica.org/programs/getcalfresh, "Overcoming Barriers...verifications". 확인 2026-06-22.

## 사례 4. Atlanta Fed CLIFF — "소득 오르면 혜택 끊김"을 미리 경고하는 계산기

- **무엇**: 소득/임금 인상 시 공공혜택 자격 상실('benefits cliff, 복지 절벽')이 발생하는 지점을 **사전 시뮬레이션**하는 도구 묶음(CLIFF Snapshot/Dashboard/Planner).
- **핵심 메커니즘**: 소득·세금·세액공제·혜택의 상호작용을 모델링 → "이만큼 더 벌면 어떤 혜택이 언제 끊긴다"를 **변화 전에** 시각화. 즉 자격변동을 *사후 통보*가 아니라 *사전 예측*으로 전환.
- **효과 근거**: 3개 주 파일럿 및 전국 파일럿 평가 존재(Atlanta Fed). 설문상 "의사결정에 도움" 응답(정량 RCT 효과치는 미확인 — 코칭 도입 평가 중심).
- **한국 이식 → K1 (가장 직접)**: **K1 자격변동 조기경보의 핵심 엔진**. "상황 변화(소득/가구원/근로)가 자격을 어떻게 바꾸는지"를 *변동 발생 전/직후* 예측해 경고. 한국 기초생활보장의 소득인정액·부양의무 기준에 같은 로직 적용 가능.
- **출처/확인**: atlantafed.org/.../cliff-tool, "Benefits Cliffs Coaching...National Pilot"(2024). 확인 2026-06-22.

## 사례 5. 자격 스크리닝 도구 — UK Turn2us / 호주 Payment & Service Finder

- **무엇**: 몇 가지 상황 질문에 답하면 **받을 수 있는 혜택·예상 금액·신청 방법**을 한 번에 보여주는 스크리닝 도구.
  - **UK Turn2us / entitledto**: 질문 시퀀스 → UC·Pension Credit·Housing Benefit 등 가능 혜택 + 금액 + 신청법.
  - **호주 Services Australia(Centrelink) Payment and Service Finder**: 상황 입력 → 받을 수 있는 지급·서비스 목록 + 신청 링크. **"지금이 아니라 곧 될 상황(예: 곧 연금 연령)으로 입력하라"** 안내 — 미래 자격 사전 점검 지원.
- **핵심 메커니즘**: "신청 기회를 놓치는" 문제를 **사전 자격 발견(eligibility discovery)** 으로 해결. 당사자가 "내가 뭘 받을 수 있는지"를 모르는 정보격차를 질문-기반 매칭으로 메움.
- **효과 근거**: 도입·운영 사실 확인(공식 사이트). 정량 효과치는 미확인.
- **한국 이식 → K1/K2 보조**: 자격변동 시 "이제 새로 받을 수 있는 것 / 잃을 것"을 함께 제시. 호주의 "미래 상황으로 입력" 패턴은 **K1 조기경보가 '곧 닥칠 변동'을 선제 안내**하는 UX 근거.
- **출처/확인**: benefits-calculator.turn2us.org.uk, centrelink.gov.au/apps/clkonline_cof/payment-service-finder. 확인 2026-06-22.

## 사례 6. UK Universal Credit 상황변경 신고(Change of Circumstances)

- **무엇**: 수급자가 소득·가구 등 변동을 **즉시 직접 신고**하도록 한 제도. 계정에서 'Report a change' 선택.
- **핵심 메커니즘 (반면교사)**: 신고 책임이 **전적으로 당사자**에게 있음 → 늦으면 과지급·환수, 안 하면 벌금·법적 조치. 어르신·디지털 취약층에 **부담 전가형** 모델.
- **효과 근거**: 제도 운영 사실 확인. (이건 "이렇게 하면 안 된다"의 사례로 활용)
- **한국 시사점 → 차별점**: 사례 1(ex-parte, 정부가 먼저 점검)과 **정반대**. 한국 어르신 대상이라면 UK식 '당사자 신고 의무'가 아니라 **사례 1+2+4의 "시스템이 먼저 감지·알림·예측" 모델**이 맞다는 근거. 우리 솔루션의 *왜 이게 필요한가* 대비 프레임.
- **출처/확인**: gov.uk/universal-credit/changes-of-circumstances, turn2us.org.uk(UC 변경 신고). 확인 2026-06-22.

## 종합: 우리 3킥 ↔ 해외 메커니즘 매핑

| 킥 | 매핑되는 해외 메커니즘 | 검증 강도 |
|---|---|---|
| **K1 자격변동 조기경보** | 사례 4 CLIFF(변동 사전 예측) + 사례 1 ex-parte(데이터로 선제 점검) + 사례 5 호주 '미래 상황 입력' | 메커니즘 강함, K1 핵심 RCT 효과치는 ex-parte(사례1)에서 차용 |
| **K2 갱신·서류 체크리스트 타임라인** | 사례 3 GetCalFresh(진행상황+서류 가시화) + 사례 2 OES 업로더(서류 마찰 제거) + 사례 1(실패분만 요청) | **RCT 근거 가장 탄탄** (인터뷰 +10.7%p, 탈락 -1.1%p) |
| **K3 어르신 음성 점검(판정은 코드)** | 사례 2 텍스트 nudge를 **음성 채널로 치환** + 사례 3 채널 선호 기반 알림 | nudge 효과는 검증, '음성' 형태 자체는 해외 직접사례 미확인 = **우리 차별점** |

### 한국에 아직 없어 보이는 가장 강한 이식 후보 (차별점의 씨앗)

1. **Ex-parte(데이터 선제 자동 점검) 로직 — 가장 강력.** 한국 갱신은 여전히 당사자 서류 제출 중심. "정부 보유 데이터로 먼저 판정하고, 통과 못한 사람만 서류 요청"은 효과(+12~28%p)가 정량 검증됐고 한국에 본격 적용 안 됨. → **K1·K2의 뼈대.**
2. **변동 *사전* 예측(CLIFF형)** — 한국은 자격 상실을 대개 사후 통보. "소득/가구 변화 시 무엇이 언제 끊기는지" 변동 전 경고는 신선. → **K1.**
3. **음성 nudge** — 검증된 문자 리마인더 효과(인터뷰 +10.7%p)를 **어르신 친화 음성**으로 옮긴 형태는 해외에서도 직접사례 미확인 → 우리만의 변형. → **K3.**

### 데이터 정직성 메모
- 정량 RCT 효과치 확보: 사례 1(ex-parte +12~28%p), 사례 2(텍스트 +10.7%p / -1.1%p). 이 둘이 발표용 "숫자 근거".
- 사례 4·5는 **메커니즘·운영은 확인, 정량 효과는 미확인** — 발표 시 "효과 수치 있음"으로 과장 금지.
- 사례 6은 효과사례가 아니라 **반면교사(부담 전가)** 로만 사용.
