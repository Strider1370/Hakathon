# 00. 통합 리서치 보고서 — 수급 자격 변동·갱신·서류 점검 서비스

> 5개 병렬 트랙(A JTBD · B 국내 · C 해외 · D 데이터/구현 · E 심사비판)을 통합. 확인 시점 2026-06-22.
> 개별 보고서: `track-a-jtbd.md` · `track-b-domestic.md` · `track-c-overseas.md` · `track-d-feasibility.md` · `track-e-judge.md`

## 0. 한 줄 결론

**받을 것을 알려주는 서비스는 많지만, "받던 게 끊길 위험"을 미리 경고하는 서비스는 국내에 없다.** 우리는 그 방향(받기 → 끊김)을 뒤집고, 어르신이 **말로** 점검하게 한다. 판정은 코드가, 해석은 AI가.

## 1. 5트랙 수렴 — 킥 우선순위

| 트랙 | 1순위 근거 | 킥 순위 |
|---|---|---|
| A JTBD (증거 무게) | 환수 5년 1,140억 + 가산세·지급제한, 원인=변동 미신고 | **K1** > K2 > K3 |
| B 국내 (공백) | "끊길 위험 경고"는 어느 서비스도 안 함(전부 상향 안내) | **K1** > K3 > K2 |
| C 해외 (메커니즘) | Ex-parte 자동갱신(+28%p), CLIFF 사전예측 | K1·K2 뼈대, K3=차별점 |
| D 데이터 (현실성) | 빠른 데모는 K2, 결정론 룰은 K1(손큐레이션) | K2 중 / K1 중~상 / K3 상 |
| E 심사 (생존력) | AI Leverage 진짜인 유일 지점=K3, 기억문장 최강=K1 | **K3** > K1 > K2 |

**수렴 결론: K1(자격변동 조기경보)을 가치의 코어, K3(어르신 음성)을 전달 레이어, K2(갱신·서류 타임라인)를 후속 액션 엔진으로 결합.** 5트랙 중 4트랙이 K1을 1~2위, E는 K3을 1위로 두는데 — 셋을 한 시나리오로 엮으면 각자의 약점이 서로 가려진다(아래 §4).

## 2. 문제의 무게 (발표용 숫자 — 출처 있는 것만)

- **환수 최근 5년 ~1,140억 원** 부적격 지급(기초생활보장 1,094.8억 등), 환수 시 **가산세 + 2~5년 지급제한**. 원인 핵심은 고의 부정이 아니라 **소득·재산·가구 변동 미신고/미인지**. (나라살림연구소·복지부 확인조사 보도자료)
- **에너지바우처 연 단위 신청제** — 기간 놓치면 그 해 못 받음. **장애 재심사 등급하락 30%·신청률 46%**. (에너지공단·홍성신문)
- **고령층 디지털정보화 70.7%**(취약계층 최저), 역량 65.1% → "왜 본인이 못 챙기나"의 근거. (2023 디지털정보격차 실태조사)
- 해외 정량(발표 인용 가능): **Medicaid ex-parte 갱신율 +28%p / 절차상 탈락 −9%p**(Health Affairs 2025), **텍스트 리마인더 인터뷰완료 +10.7%p**(Boulder County RCT).

> 미확인(날조 금지): 환수 원인별 건수 분해, 갱신 미준수 단절 건수, 서류 미비 반려율 전국 통계는 미확보 → "미확인" 표기.

## 3. 경쟁 지형 — 무엇이 이미 있나 (중복 회피)

- **복지로 복지멤버십**: 163종 자동 안내(카톡 확대). 한계 = 사전가입 필수 · 신청은 본인 책임 · **'받을 것(상향)'만**, 끊김 경고 없음.
- **보조금24 / 국민비서 구삐**: 신규 신청·마감 알림 중심. 복지급여 통합 '끊김 경보'는 아님.
- **행복e음**: 자격 판정 로직이 **공무원 백오피스에 갇힘** — 당사자에게 노출 안 됨. (← 우리가 당사자 코드로 재현하는 지점)
- **민간(토스 등)**: 토스 '숨은 지원금'은 2024-08 종료. 전부 보조금24 API 재활용 + 앱 능숙 전제 → 어르신 배제.

**3대 공백(국내 직접 경쟁자 없음):** ① 끊김 사전경보 ② 갱신+서류 누락 통합 타임라인 ③ 어르신 음성/대면 점검.

## 4. 해외에서 이식할 검증된 메커니즘

1. **Ex-parte 자동갱신**(US Medicaid) — 정부 보유 데이터로 선제 판정, 통과면 무서류 갱신. → K1/K2 뼈대. 정량 최강.
2. **CLIFF 사전 시뮬레이션**(Atlanta Fed) — 소득 변동이 *언제* 혜택을 끊는지 미리 예측. → K1 엔진.
3. **텍스트 리마인더 nudge RCT** — 재인증율 상승 검증. → 어르신 친화 **음성 nudge로 이식 = K3(해외 직접사례 없음 = 우리 차별점)**.
4. 보조: Turn2us·Payment Finder(질문 기반 사전 자격), GetCalFresh(남은 서류·마감 가시화).
5. 반면교사: UK UC '상황변경 신고 책임 당사자 전가' — 어르신엔 부적합.

## 5. 데이터·구현 현실 (D트랙 실측)

- **정규화된 정부 데이터셋 없음(확정).** gov24 `신청기한`=자유서술 2,510종, 복지로엔 신청기한 필드 자체 없음. 선정기준=산문.
- **결론: 대표 10~12개를 권위 URL에서 손 큐레이션** → `curated-benefits.json`에 `eligibility`(코드 술어)·`renewMonth`·`deadline`·`documents[]` 슬롯. 모집단(검색)은 전수 16,000건 유지.
- 재사용 자산: `@krds-ui/core`, `lib/regions.ts`, `lib/llm.ts`+`/api/llm`(단 키 주입 갭), Header/Footer. `page.tsx`는 빈 플레이스홀더 → 폼·결과 카드 신규.
- K3 음성: `@openai/agents-realtime` 미설치, WebRTC·마이크·키모달 신규 → **난이도 상, 타이핑+녹화 폴백 필수.**

## 6. 심사 방어 (E트랙) — 발표에서 선제 제거할 의심

- **"복지멤버십이 이미 하잖아?"** → 한 슬라이드로 **방향 차이(받기 vs 끊김)** 못박아 죽임.
- **"왜 AI? 규칙이면 되잖아?"** → 비정형 음성/상황 서술 → 개념 변환은 규칙 불가(K3). **판정은 코드**(Guardrail).
- **"자격 판정 틀리면 책임은?"** → 단정 금지, "잠재 위험 + 공식 URL 확인" + 사람 최종판단 명시.
- **"어르신이 음성 진짜 쓰나?"** → 타이핑 불가 어르신의 유일 입력 + 라이브 음성 데모.
- **데모 시간 위험**: 5분에 셋 다 라이브면 쪼개짐 → **K3→K1만 라이브, K2는 같은 화면에 결과 1컷**.
