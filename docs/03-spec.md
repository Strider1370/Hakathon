# 03. 스펙 (Spec) — 확정 범위

> 체크리스트 03 산출물. 확정된 것만. 데모 시나리오 1개 + 화면 입출력 구체화 + 음성 범위 + 범위 컷.
> 인사이트: `docs/02-concept.md`. 데이터: `data/welfare/curated-benefits.json`(스크랩 병합 예정).

## 1. Answer First
**어르신(수급 당사자)** 을 위해 / **받던 복지가 끊길 위험과 갱신·서류 누락** 을 / **말로 상황을 얘기하면 코드가 점검해 D-day·할 일·서류로 안내** 한다.

## 2. Problem (압축)
- 환수 5년 ~1,140억(가산세·지급제한), 원인=소득·재산·가구 변동 미신고/미인지.
- 기존 서비스는 전부 *받을 것(상향)* 안내 → *끊김*은 빈칸. 어르신은 타이핑·로그인·앱에서 배제.

## 3. AI Leverage (역할 명시)
- AI 없이 가능: 혜택 카탈로그 검색.
- AI로 빨라짐: 자격 산문 요약.
- **AI라서 처음 가능**: 비정형 음성 상황 서술 → 점검 슬롯 값 추출.
- **AI가 안 하는 것**: 끊김/자격 *판정*은 결정론적 코드(`rules.eligibility`). AI는 단정·판정 금지. 최종 판단=사람. 출처=공식 URL 화이트리스트.

## 4. Demonstration — 단일 시나리오 (Wow 포함)
**페르소나(가상·익명):** 김순자, 75세, 기초연금 수급 중. 최근 **요양보호사 일을 시작**(근로소득 발생) + **혼자 살다 아들과 합가**(가구원 변동).

1. 첫 화면 — 큰 글씨 "지금 상황을 말씀해 주세요" + 🎤 버튼.
2. 음성: *"내가 이번에 일을 좀 시작했고, 아들이랑 같이 살게 됐어."*
3. AI가 상황 해석 → 슬롯 채움(`근로소득 발생=true`, `가구원 변동=합가`). 화면에 **읽어주기 확인**("일 시작·합가, 맞나요?").
4. **코드가 판정** → 결과 카드 3종 등장:
   - 🔴 **끊김 경고**: "기초연금 — 가구소득·합가로 *선정기준액 초과* 위험. **지금 신고 안 하면 환수+가산세**. D-?"
   - 🟡 **갱신 임박**: "에너지바우처 — 신청기간 D-12, 놓치면 올해 못 받음."
   - 🟢 **유지**: "의료급여 — 변동 영향 없음."
5. 🔴 카드 탭 → **조치 안내**: 해야 할 일(변동 신고) · 필요 서류 체크리스트 · 관할(읍면동/복지로) · **공식 URL**.

**Wow Moment:** 말 한마디에 빨간 끊김 경고가 *먼저* 뜬다 — "받기"가 아니라 "끊김"을 짚는 화면.
**성공 기준:** 음성→슬롯→빨간 경고 카드가 라이브로 보이고, 조치/서류/URL이 화면에 존재.
**실패 시 폴백:** 음성 실패 → 텍스트 입력칩(연령·소득변동·가구변동·받는혜택) 또는 사전 녹화 재생.

## 5. 화면 입출력 (구체)
**입력 요소**
- 🎤 음성 입력(주) / 텍스트 입력칩(폴백): `연령`, `받는 혜택(체크: 기초연금·의료급여·에너지바우처…)`, `최근 변동(소득↑·재산↑·가구변동·없음)`.
- 지역 선택(`lib/regions.ts` 시도→시군구) — 관할 안내용.

**출력 요소**
- 결과 카드(상태색 🔴🟡🟢) — 혜택명 · 상태문구 · 위험사유 · D-day.
- 조치 패널 — 할 일 1~3단계 · 서류 체크리스트(`documents[]`) · 신청 한 줄 · 공식 URL 버튼.
- Guardrail 배지 — "잠재 위험 추정 · 최종 확인은 관할/공식 페이지" (단정 금지).

## 6. 음성 범위 결정 (★확인 필요)
- **권장(현실안)**: 음성은 **헤드라인 입력**으로 라이브 시연하되, **타이핑 폴백 + 사전 녹화**를 1급 백업으로 둔다. 발표장 소음·WebRTC·마이크 권한 리스크(D트랙: 난이도 상) 때문.
- 음성 엔진: `gpt-realtime`(키=런타임 모달, 없으면 텍스트 폴백). `@openai/agents-realtime` 신규 설치 필요.
- 대안(보수안): 음성 빼고 **텍스트 칩 입력**만 — 구현·시연 리스크 최소, 단 K3 차별점 약화.

## 7. 판정 엔진 (결정론) — 신뢰 핵심
- `rules.eligibility` 코드 술어 + 사용자 슬롯값 → **끊김/유지/갱신 판정**. AI 자유분류 금지.
- 모집단 검색은 전수 16,000건(gov24+복지), **정밀 판정은 curated 10~12개** 한정.
- 판정 불가/근거 약함 → "확인 필요" + 공식 URL.

## 8. 범위 컷 (MVP → 확장)
**MVP(데모 핵심경로):** 첫화면(입력) → 판정 → 결과 카드 3종 → 조치 패널 1개. curated 6~8개로도 데모 성립.
**확장:** 음성 라이브 · 카드 더보기(전수 검색) · 지도(관할 복지관, `facilities`/대피소 좌표) · D-day 캘린더.
**드롭 후보:** 로그인·실계정 연동·푸시 알림 백엔드(발표선 'mocked' 명시).

## 9. Guardrail / 한계
- Soft: 일부 혜택 미수록(대표 큐레이션). Bounded: 자격은 추정, 공식 확인 필요. Hard: 환수·법적 판단은 관할 기관.
- 개인정보: 실데이터 금지(가상 페르소나). 민감정보 외부 LLM 전송 최소화.
