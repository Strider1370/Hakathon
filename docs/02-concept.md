## B. 레드팀 (E트랙 수행 결과 반영)

가장 강한 의심 = **"복지로 복지멤버십이 이미 하잖아?"**
- 방어: 복지멤버십은 **받을 것(상향)만** 안내, **끊김 경고는 안 함** → *방향이 반대*. 한 슬라이드로 못박음.

기타 선제 방어: "왜 AI?"→비정형 음성→개념 변환은 규칙 불가(I3) / "틀리면 책임?"→판정은 코드 + "잠재 위험 + 공식 URL 확인" + 사람 최종판단 / "어르신이 음성 쓰나?"→타이핑 불가 어르신의 유일 입력 + 라이브 데모.

자기검증: I1은 수사가 아니라 환수 통계로 뒷받침됨 ✓ · Solution을 Insight로 포장 안 함(관점=방향 전환) ✓ · AI 연결 억지 아님(I3 음성 해석) ✓.

## C. 확정 인사이트

**I1을 코어 + I3을 전달 + I4를 후속 액션으로 결합.**

- **Reframing(확정)**: "복지는 받는 순간이 아니라 *유지하는 동안* 끊긴다. 끊기기 전에, 말로 점검한다."

## D. Concept

- **한 문장 피치**: 어르신이 말로 상황을 얘기하면, 받던 혜택 중 *끊길 위험*을 미리 찾아 D-day와 해야 할 일을 짚어주는 점검 비서.
- **킥(차별점)**: 방향을 뒤집음 — 모두가 *받기*를 알려줄 때 우리는 *끊김*을 경고. (Insight·Narrative 차별화 = 최상위 등급)
- **Wow Moment**: 말로 "나 이번에 일 시작했어요" → 화면에 **빨간 경고 카드**: "근로장려금 소득기준 초과 위험 · D-12 · 지금 △△ 하세요". (판정=코드, 해석=AI)
- **마지막 문장**: "받는 걸 돕는 복지는 많습니다. 끊기지 않게 지키는 복지는, 여기서 시작합니다."

## E. AI Leverage (신뢰 원칙 §1.3)

- AI 없이도 가능: 혜택 카탈로그 검색.
- AI로 빨라짐: 복잡한 자격 산문 요약.
- **AI라서 처음 가능**: 어르신의 비정형 음성 상황 서술 → 점검 항목으로 변환(규칙 불가).
- **AI가 안 하는 것**: 자격 끊김 *판정*은 결정론적 코드. AI는 단정하지 않음. 최종 판단=사람.
