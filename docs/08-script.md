# 08. 발표 대본 (5분)

> 인사이트: `docs/02-concept.md` · 구현 상태: `docs/manifest.md`.
> 흐름: 답먼저 → 문제 → 인사이트 → 솔루션 → **라이브 데모(2분)** → 메커니즘 → 임팩트 → 한계/가드레일 → 클로징.
> 화면 카피(deck.json 슬롯)는 **사람이 최종 확정**. 아래 스피커노트는 AI 초안(애드립 가능).

## 시간 배분
| 구간 | 슬라이드 | 시간 |
|---|---|---|
| 답먼저 + 문제 | 1·2 | ~45s |
| 인사이트 | 3·4 | ~40s |
| 솔루션 | 5 | ~20s |
| **라이브 데모 + Wow** | (앱) | **~120s** |

| 메커니즘 | 7 | ~25s |
| 임팩트 | 8 | ~20s |
| 한계/가드레일 | 9 | ~25s |
| 클로징 | 10 | ~20s |

## 슬라이드별 (메시지 + 스피커노트)

**1. hero — 답먼저**
화면: "받던 복지, 끊기기 전에" / "어르신이 말로 점검하는 자격 변동 비서"
노트: "복지의 진짜 문제는 못 받는 게 아니라, 받던 게 끊기는 겁니다. 우리는 그걸 끊기기 전에, 말로 점검합니다."

**2. problem-flow — 문제**
화면: 소득·가구 바뀜→자격 변동 / 갱신·서류 놓침→그대로 끊김 / 본인은 환수 통지로 안다
노트: "소득이 늘거나 가족과 합치면 자격이 바뀝니다. 본인은 모르고 지내다, 환수 통지로 압니다. 최근 5년 부적격 지급이 1,140억, 가산세에 지급 제한까지 붙습니다."

**3. contrast — 인사이트(경쟁 대비)**
화면: 기존(복지로·보조금24·토스)=받을 것 안내·신규 신청·앱 전제 / 우리=끊길 위험 경고·유지 갱신·말로 점검
노트: "받을 것 찾아주는 서비스는 많습니다. 복지멤버십도 그렇죠. 그런데 전부 방향이 '받기'입니다. 끊길 위험을 경고하는 건 없습니다. 방향을 뒤집었습니다."

**4. insight-statement — 한 문장**
화면: "복지는 유지하는 동안 끊긴다" / "끊기기 전에, 말로 점검한다"
노트: (한 박자 쉬고) "복지는 받는 순간이 아니라, 유지하는 동안 끊깁니다."

**5. product-overview — 솔루션**
화면: 말로 문진→받는 혜택 발견 / 변동 점검→끊길 위험 / 본인인증→추정을 정확으로
노트: "세 단계입니다. 말로 물어 받는 걸 찾고, 변동을 점검해 끊길 위험을 짚고, 본인인증하면 추정이 정확한 숫자로 바뀝니다. 직접 보시죠."

**→ 라이브 데모 (앱, ~2분)** — 백업: `.shots/` 단계별 스크린샷
1. "🎤 음성으로 시작" → 도우미가 음성으로 "65세 넘으셨나요?" (자막 동시)
2. 말로 답 → 받는 혜택 발견(기초연금·의료급여)
3. "이어서 끊길 위험" → "일 시작·합가" → **🔴 기초연금·의료급여 끊길 위험 + D-day + 서류**
4. "본인인증" → (목업) → 소득인정액 238만/선정 247만, **D-173**

**6. demo-callout — 데모 리캡**
화면: 말 한마디 / 빨간 끊김 경고 / D-day·할 일·공식 URL
노트: "말 한마디에 끊길 위험이 먼저 떴습니다. 받기가 아니라, 끊김을요."

**7. architecture — 메커니즘**
화면: 음성 이해→슬롯 추출(strict 함수호출) / 결정론 코드(cutoffTriggers 매칭) / 공식 출처 카드 / 결과: 환각 없는 판정
노트: "AI는 듣고 값만 뽑습니다. 자격 판정은 결정론 코드가 합니다. 그래서 똑똑함이 아니라 정직함이 차별점입니다 — 틀려도 엉뚱한 결과가 안 나갑니다."

**8. big-number — 임팩트**
화면: "1,140억" / 변동 미신고 환수 5년 · 해외 ex-parte 자동갱신 +28%p
노트: "문제의 크기는 5년 1,140억. 해외에선 정부 데이터로 미리 점검하는 ex-parte 방식이 갱신율을 28%p 올렸습니다(Health Affairs 2025). 우리가 그걸 당사자 편에서 합니다."

**9. limitation-guardrail — 한계/가드레일**
화면: 한계(자격은 추정·대표 12개·음성 폴백) / 가드레일(판정은 코드·단정 금지·공공마이데이터 목업)
노트: "정직하게 짚습니다. 로그인 전 자격은 추정이고, 정확한 소득인정액은 본인인증이 필요합니다. 정부 연계 부분은 공공마이데이터를 가정한 목업입니다."

**10. closing**
화면: "받는 걸 돕는 복지는 많습니다" / "끊기지 않게 지키는 복지는, 여기서 시작합니다"
노트: "받는 걸 돕는 복지는 많습니다. 끊기지 않게 지키는 복지는, 여기서 시작합니다. 감사합니다."

## Q&A — 심사위원 공격질문 10개 + 답

1. **"복지로 복지멤버십이 이미 하잖아요?"** → 복지멤버십은 '받을 것(상향)'만 안내합니다. '끊길 위험'은 안 줍니다 — 방향이 반대입니다. 게다가 사전가입·본인 신청 책임이라 어르신 도달이 깨집니다.
2. **"왜 AI인가요? 규칙 테이블이면 되잖아요?"** → 판정은 실제로 규칙(코드)입니다. AI는 어르신의 비정형 음성을 점검 항목으로 바꾸는 입력 단계에만 씁니다 — 그건 규칙으로 안 됩니다.
3. **"자격 판정이 틀리면 책임은?"** → 단정하지 않습니다. "끊길 위험 + 공식 확인"으로 표시하고 최종 판단은 관할 기관입니다. 핵심 판정은 공식 출처 기반 결정론 코드라 환각이 안 끼어듭니다.
4. **"어르신이 음성을 진짜 쓰나요?"** → 타이핑이 가장 큰 장벽인 분들께 음성은 유일한 입력입니다. 자막을 동시에 띄워 이중 확인하고, 안 되면 탭으로 폴백합니다.
5. **"소득인정액을 어떻게 아나요?"** → 모릅니다, 본인도. 그래서 로그인 전엔 추정으로 정직하게 표시하고, 본인인증(공공마이데이터) 후 정확한 값으로 바꿉니다. 시연의 연계는 목업입니다.
6. **"데이터는 진짜인가요?"** → 보조금24 10,957 + 복지 5,021건이 모집단이고, 대표 12개는 보건복지부·국세청·법령정보 등 공식 출처에서 자격·갱신·서류를 큐레이션했습니다(출처 URL 포함).
7. **"개인정보는요?"** → 시연은 가상 페르소나입니다. 실서비스는 본인인증 동의 기반 공공마이데이터, 민감정보는 외부 LLM에 보내지 않습니다.
8. **"갱신 날짜(D-day)는 어디서?"** → 제도별 법령·고시에서 큐레이션했습니다. 카탈로그엔 일관된 날짜가 없어 대표 혜택만 정밀화했고, 그 한계를 그대로 표시합니다.
9. **"확장 가능한가요?"** → 모집단은 이미 전수입니다. curated 룰을 늘리고, 공공마이데이터 실연계를 붙이고, 국민비서 알림 채널로 푸시하면 됩니다.
10. **"음성이 발표장에서 안 되면?"** → 타이핑 문진이 1급 폴백이고, 단계별 스크린샷 백업이 있습니다. VAD 민감도도 현장에서 조절 가능합니다.
